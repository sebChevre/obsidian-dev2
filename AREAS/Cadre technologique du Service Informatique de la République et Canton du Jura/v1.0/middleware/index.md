# :material-middleware: Domaine Middleware
> Ensemble des **services logiques transverses et des runtimes applicatifs** sur lesquels s’appuient les solutions métiers.

### Finalité
Offrir des **services communs** et des **capacités logicielles réutilisables** (identité, orchestration, intégration, exécution applicative).

### Critères d’appartenance
- Fournit un **service logique ou fonctionnel partagé** entre plusieurs applications ou métiers.  
- Consomme les **ressources de l’infrastructure** mais **alimente** les plateformes métiers.  
- N’est **pas dédié à un métier spécifique**.  
- Peut être **mutualisé** à l’échelle du SI.  
- A une **responsabilité fonctionnelle transversale** (identité, exécution, intégration, automatisation).  

**Exemples:** 
Microsoft Active Directory, Adaxes, .NET Runtime, Java SE, BizTalk, Apache Airflow, API Gateway, outils CI/CD.
