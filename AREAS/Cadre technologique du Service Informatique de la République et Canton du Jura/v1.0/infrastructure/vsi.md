# Virtual Server Infrastructure
<!-- description/short {start} -->
Virtual Server Infrastructure (VSI) est une architecture d’infrastructure virtuelle permettant de conjuguer consolidation, flexibilité et gestion centralisée des serveurs physiques et virtuels sur un même pool de ressources.
<!-- description/short {end} -->

## Description
**Virtual Server Infrastructure (VSI)** désigne un environnement où les ressources physiques (serveurs, stockage, réseau) sont virtualisées pour héberger plusieurs machines virtuelles sur un même matériel, permettant ainsi une meilleure utilisation, une flexibilité accrue et une gestion centralisée.  
Le modèle VSI permet de provisionner rapidement des « instances serveur », d’ajuster dynamiquement les capacités et de consolider les infrastructures distantes ou virtualisées.  
Il offre souvent une architecture « hyper-convergée » ou « cloud privé on-premise », intégrant de l’automatisation, du pooling de ressources et des stratégies de résilience.  
Dans le contexte de la RCJU, le VSI peut servir de socle pour l’hébergement des systèmes critiques, en assurant performance, continuité et simplification des opérations informatiques.  
Ce choix architectural s’inscrit dans la transformation vers un modèle d’infrastructure plus agile, standardisé et maîtrisé.
